VALE Web App (Fase 1) — Brief para Desarrollo
1) Contexto y decisión
Existe un MVP en WordPress (vale.club) que no se escalará: se mantiene solo como landing/captación de leads.


Se construirá una web app a medida (código custom) como base escalable para:


directorio de empresas


perfiles profesionales


filtros y búsqueda rápida (2–3 clics)


sistema de reseñas bidireccional y validado


futura conversión a app nativa


Objetivo de entrega: plataforma lista entre el 10 y el 15 de marzo (aprox.) para onboarding de ~10 clientes confirmados.

2) Alcance Fase 1 (MVP real escalable)
2.1 Journey usuario (UX clave)
Mobile-first. El usuario debe poder encontrar un servicio en 2–3 clics:
Home: buscador + áreas/categorías principales


Listado: resultados + filtros


Perfil empresa: contacto y detalles


Opcional (si se llega): vista mapa.
Requisito UX explícito: no navegar “15 pantallas”. Prioridad máxima: simple, rápido, profesional.

Cualquier cambio de datos de contacto implica validación por email para confirmar



3.4 Sistema de reseñas bidireccional (core feature) (Dont do we will do it in phase 2)
Objetivo: proteger a empresas de malos clientes y dar confianza a usuarios.
Flujos:
Usuario → Empresa: deja reseña (rating + texto)


Empresa → Usuario: puede dejar reseña al cliente (o al menos responder)


Restricción importante: si el contacto ocurre por WhatsApp fuera de la plataforma, hay que validar que existió servicio antes de permitir reseña.
Validación (MVP):
Opción A: Empresa confirma “ok asistió” y habilita reseña


Opción B: usuario sube “proof” (foto ticket/recibo/factura) para habilitar reseña


Opción C (futuro / opcional): IA hace match de evidencia (OCR ticket / verificación imagen vs fotos del local)


Para Fase 1, implementar A + B con revisión manual (admin) si hace falta; IA puede quedar como “placeholder”.

Sugerencia técnica: construir ya entidad “User” (cliente) aunque el onboarding sea ligero.
